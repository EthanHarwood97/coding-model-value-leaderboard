How Rankings Work

Click to expand methodology

This leaderboard follows the research consensus on which LLM leaderboards are most trustworthy. We blend multiple high-signal sources rather than relying on any single benchmark.

Coding Quality Score (0–100) is a weighted composite: 35% SWE-bench Pro, 30% Aider Polyglot, 20% LiveCodeBench, 15% LiveBench. Models missing any component get a proportional score from available data. This rewards models that perform well across diverse coding benchmarks rather than gaming one.

$/Quality Point = Output price per million tokens ÷ Coding Quality Score (normalized to 0–100). Lower = better value. This is the most holistic value metric on the page.

SWE-bench Pro is the hardest public coding benchmark — real GitHub issues, multi-file fixes, no human hinting. (source)

Aider Polyglot is the gold standard for coding-specific LLM evaluation. It tests 225 real Exercism exercises across Python, Go, Rust, JavaScript, C++, and Java — measuring how well a model edits code without breaking the build. (source)

AA Coding Index is Artificial Analysis's composite intelligence score, built from MMLU-Pro, GPQA Diamond, MATH-500, HumanEval, and more. Broad coverage across 80+ models. (source)

SWE-bench Verified is the human-validated subset (~500 issues). Useful but frontier models cluster tightly here.

LiveCodeBench measures competitive programming with monthly question refreshes to prevent contamination.

LiveBench is a contamination-frozen benchmark with a global average across math, reasoning, language, and coding tasks. Uses 2026 snapshots from livebench.ai. (source via whichllm)

Terminal-Bench 2.1 measures agentic CLI performance — directly relevant for terminal-based tools.

$/Pro Point = Output price per million tokens ÷ SWE-bench Pro score. Lower = better value.

Speed is tokens/sec from Artificial Analysis benchmarks where available.

Prices are list rates per million tokens (USD). Cache-hit rates shown in the data file but not the table.

License indicates whether weights are available (MIT/Apache/etc.) or proprietary API-only.

Benchmarks sourced from independent evaluators. Some scores are self-reported by providers — see data/models.json source URLs for provenance. What we avoid: vendor cherry-picked charts, single-benchmark boards (MMLU alone is saturated), and Reddit polls.
